Stash Managed Catalog API (1.0.0)

Download OpenAPI specification:

API Support: help@stash.gg URL: https://docs.stash.gg/

Read-only API for querying the Stash managed product catalog. Designed for server-to-server communication between partner backends and Stash services.

Base URLs

Environment Base URL
Test https://test-api.stash.gg
Production https://api.stash.gg

All endpoint paths below are relative to the base URL. For example, to list products in the test environment:

GET https://test-api.stash.gg/sdk/server/catalogue/products

Authentication

All requests must include your API key in the X-Stash-Api-Key header. The shop is inferred from the key — no shop ID is required in the path.

Setup

  1. Go to Stash Studio > Project Settings > API Secrets
  2. Click Generate Secret to create a new key
  3. Use the generated key as the value for the X-Stash-Api-Key header on every request
  4. Store the key securely — treat it like a password

Example (curl)

curl -H "X-Stash-Api-Key: YOUR_API_KEY" \
  "https://test-api.stash.gg/sdk/server/catalogue/products"

Managed Catalog

Read-only endpoints for querying the managed product catalog

List published products

Retrieves a paginated list of published products with active offers for the shop associated with the API key. Results are ordered by creation date (most recent first). Only published products whose offer window contains the current time are returned; drafts, deleted products, and offers outside their schedule are excluded.

Authorizations:
apiKey
query Parameters
limit
integer <int64>

Maximum results per page. Defaults to 50. Maximum allowed value is 100.

offset
integer <int64>

Pagination offset. Use next_offset from the previous response to fetch the next page. Defaults to 0.

Responses

/sdk/server/catalogue/products

Get a single published product

Retrieves a single published product with an active offer by its partner-defined product ID (e.g., SKU) for the shop associated with the API key. Returns NOT_FOUND if the product does not exist, is not published, or its offer is outside the current schedule.

Authorizations:
apiKey
path Parameters
productId
required
string

Partner-defined product identifier (e.g., SKU)

Responses

/sdk/server/catalogue/products/{productId}
